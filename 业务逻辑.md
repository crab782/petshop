### 📋 订单存储方式
**服务订单和商品订单是完全分开存放的**，使用不同的数据库表和业务逻辑：

1. **服务订单（Appointment）**
   - 表：`appointment`
   - 用于预约店铺提供的服务（宠物洗澡、美容、寄养等）
   - 一个预约对应一个服务

2. **商品订单（ProductOrder）**
   - 主表：`product_order`
   - 明细表：`product_order_item`
   - 用于购买店铺销售的商品（宠物食品、用品等）
   - 一个订单可以包含多个商品

---

### 🔄 服务订单业务流程

**状态流转**：
```
用户预约 → pending（待处理）
    ↓
商家确认 → confirmed（已确认）
    ↓
服务完成 → completed（已完成）
```

**状态更新规则**：
- `pending` → `confirmed`（商家确认）
- `pending` → `cancelled`（取消预约）
- `confirmed` → `completed`（完成服务）
- `confirmed` → `cancelled`（取消预约）
- 已完成或已取消的预约不能修改

**关键业务逻辑**：
- 必须指定宠物和预约时间
- 只有商家可以确认和完成服务
- 用户和商家都可以取消预约

---

### 📦 商品订单业务流程

**状态流转**：
```
用户下单 → pending（待支付）
    ↓
用户支付 → paid（已支付）
    ↓
商家发货 → shipped（已发货）
    ↓
用户确认收货 → completed（已完成）
```

**状态更新规则**：
- `pending` → `paid`（用户支付）
- `pending` → `cancelled`（取消订单）
- `paid` → `shipped`（商家发货）
- `paid` → `cancelled`（退款取消）
- `shipped` → `completed`（用户确认收货）
- 已完成或已取消的订单不能修改

**关键业务逻辑**：
- 需要收货地址和支付方式
- 支持物流跟踪
- 商家发货需要填写物流信息
- 用户确认收货后订单完成
