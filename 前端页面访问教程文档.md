# 前端页面访问教程文档

## 1. 项目概述

本项目是一个宠物管理系统的前端部分，使用Vue 3 + TypeScript + Element Plus构建。项目包含三个主要角色的功能模块：

- **用户端**：普通用户可以浏览服务、预约服务、购买商品、管理个人信息等
- **商家端**：商家可以管理服务、商品、订单、预约、店铺设置等
- **平台端**：管理员可以管理用户、商家、服务、商品、公告、系统设置等

项目采用了现代化的前端技术栈，包括Vue 3的Composition API、Pinia状态管理、Vue Router路由管理等。

## 2. 项目启动指南

### 2.1 环境要求

- Node.js：^20.19.0 || >=22.12.0
- npm：随Node.js安装

### 2.2 安装依赖

```bash
# 进入前端项目目录
cd cg-vue

# 安装依赖
npm install
```

### 2.3 启动开发服务器

```bash
# 启动开发服务器
npm run dev
```

开发服务器启动后，会在控制台输出访问地址，通常为 `http://localhost:5173/`。

### 2.4 构建生产版本

```bash
# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

## 6. 路由结构分析

### 6.1 路由配置概述

项目使用Vue Router进行路由管理，路由配置位于 `src/router/index.ts` 文件中。路由结构分为三个主要部分：

1. **公共路由**：无需登录即可访问的页面，如登录、注册等
2. **用户端路由**：需要用户角色权限的页面
3. **商家端路由**：需要商家角色权限的页面
4. **平台端路由**：需要管理员角色权限的页面

### 6.2 路由守卫

当前项目已临时禁用登录验证，路由守卫代码如下：

```typescript
// 路由守卫（已禁用登录验证，仅用于调试）
router.beforeEach((to, from, next) => {
  // 临时禁用所有登录验证，方便直接访问页面进行测试
  next();
});
```

### 6.3 路由结构详解

#### 6.3.1 公共路由

```typescript
{
  path: '/',
  redirect: '/user/home',
},
{
  path: '/about',
  name: 'about',
  component: () => import('../views/AboutView.vue'),
},
{
  path: '/login',
  name: 'login',
  component: () => import('../views/login/Login.vue'),
},
{
  path: '/register',
  name: 'register',
  component: () => import('../views/login/Register.vue'),
},
{
  path: '/merchant/register',
  name: 'merchant-register',
  component: () => import('../views/merchant/Register.vue'),
},
{
  path: '/merchant/login',
  name: 'merchant-login',
  component: () => import('../views/merchant/Login.vue'),
```

## 7. 访问注意事项

### 7.1 开发环境访问

- 确保已安装Node.js环境（版本 ^20.19.0 || >=22.12.0）
- 安装依赖后启动开发服务器
- 访问地址通常为 `http://localhost:5173/`

### 7.2 路由访问

- 项目默认首页会自动重定向到 `/user/home`
- 大部分页面已配置默认参数，如服务详情页默认访问ID为1的服务
- 当前项目已临时禁用登录验证，可直接访问所有页面进行测试

### 7.3 测试数据

- 项目使用Mock数据进行模拟，无需后端服务即可查看页面效果
- 部分页面可能需要特定的URL参数才能正常显示数据

### 7.4 浏览器兼容性

- 推荐使用Chrome、Firefox、Edge等现代浏览器
- 不建议使用IE浏览器，可能存在兼容性问题

### 7.5 性能优化

- 开发环境下可能会有一些性能开销，生产环境会进行优化
- 页面加载速度可能会受到网络环境和设备性能的影响

### 7.6 其他注意事项

- 本项目为前端演示版本，部分功能可能尚未完全实现
- 实际部署时需要启用路由守卫进行权限验证
- 生产环境需要配置合适的API接口地址
