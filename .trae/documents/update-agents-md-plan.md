## 具体更新内容

### 新增 Git 操作规范
在现有的终端操作要求和系统环境说明之后，添加以下内容：

```markdown
# Git 操作规范

## 代码操作后的 Git 提交
- **执行时机**：在完成代码修改或功能实现后
- **操作步骤**：
  1. `git add .` - 添加所有修改的文件
  2. `git commit -m "[合理的提交信息]"` - 提交到当前本地分支
  3. 设置代理环境变量（如需访问 GitHub）
  4. `git push origin 1:main` - 推送到远程仓库 main 分支

## 提交信息格式
- **格式**：简短描述 + 详细说明（可选）
- **示例**：
  - `feat: 添加用户登录功能`
  - `fix: 修复订单支付bug`
  - `docs: 更新API文档`
  - `chore: 优化构建流程`

## 代理环境变量设置
当需要访问 Google 或 GitHub 时，设置以下临时环境变量：

```powershell
$env:HTTP_PROXY="http://127.0.0.1:7897"
$env:HTTPS_PROXY="http://127.0.0.1:7897"
```

## 推送操作
- **目标分支**：远程仓库的 main 分支
- **命令**：`git push origin 1:main`
- **说明**：将本地分支 1 推送到远程仓库的 main 分支
```

## 注意事项
1. 确保在推送前所有修改已正确提交
2. 确保代理环境变量设置正确，以避免网络访问问题
3. 提交信息应清晰明了，便于代码审查和版本管理
4. 推送操作可能需要输入 Git 凭据

## 风险处理
- 如果推送失败，检查网络连接和代理设置
- 如果 Git 冲突，解决冲突后重新提交
- 如果权限问题，确保有正确的 Git 权限
