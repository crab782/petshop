# 恢复前端配置到合并前状态计划

## 目标
恢复到合并分支前的配置状态：
- 前端端口：5173
- API 基础 URL：http://localhost:8080
- CORS 配置：旧的配置

## 执行步骤

### 步骤 1: 修改 .env.development 配置
- 将 `VITE_API_BASE_URL` 从 `/api` 改回 `http://localhost:8080`

### 步骤 2: 恢复 vite.config.ts 中的 host 和 port 配置（如果需要）
- 确保 host 为 127.0.0.1 或 localhost
- 确保 port 为 5173

### 步骤 3: 恢复后端 CORS 配置（如果需要）
- 移除新增的 80 端口配置
- 恢复原来的 CORS 配置

### 步骤 4: 重启前端开发服务器
- 停止当前运行的前端服务
- 启动新的前端开发服务器

### 步骤 5: 验证配置
- 确认前端配置正确
- 确认前端能够正常访问后端 API

## 注意事项
1. 需要确保所有修改与合并前的配置一致
2. 重启服务后需要验证功能正常
3. 如果使用直接连接后端的方式，不需要 Vite 代理
