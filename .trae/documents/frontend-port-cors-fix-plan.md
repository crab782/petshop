# 前端端口修改与CORS配置计划

## 项目分析

### 当前配置
- **前端**：运行在 127.0.0.1:3000，代理配置：/api → http://localhost:8080
- **后端**：运行在 8080 端口，CORS 允许的源包括：http://localhost:5173, 3000, 5174, 5175 及对应的 127.0.0.1 版本

### 目标配置
- **前端**：运行在 80 端口
- **后端**：保持 8080 端口
- **CORS**：更新配置以允许 http://localhost:80 和 http://127.0.0.1:80

## 实施计划

### 1. 修改前端配置
- **文件**：`e:\g\petshop\petshop-vue\vite.config.ts`
- **修改**：
  - 将 `port` 从 3000 改为 80
  - 保持代理配置不变

### 2. 更新后端 CORS 配置
- **文件**：`e:\g\petshop\src\main\java\com\petshop\config\SecurityConfig.java`
- **修改**：
  - 在 `setAllowedOrigins` 中添加 `http://localhost:80` 和 `http://127.0.0.1:80`

### 3. 验证配置
- **前端**：启动开发服务器，确认运行在 80 端口
- **后端**：重启后端服务
- **CORS**：测试前端能否正常访问后端 API

### 4. 测试场景
- 前端页面加载
- API 调用测试
- 登录功能测试
- 数据获取测试

## 风险评估

### 潜在问题
1. **端口权限**：80 端口可能需要管理员权限
2. **端口冲突**：80 端口可能被其他服务占用
3. **CORS 配置**：需要确保后端正确更新 CORS 配置

### 解决方案
1. **端口权限**：以管理员身份运行命令
2. **端口冲突**：检查并停止占用 80 端口的服务
3. **CORS 配置**：仔细检查配置更新是否正确

## 预期效果

- 前端运行在 http://localhost:80
- 后端运行在 http://localhost:8080
- 前端能够正常访问后端 API
- 无 CORS 跨域错误

## 实施步骤

1. 修改前端 vite.config.ts 配置
2. 更新后端 SecurityConfig.java 的 CORS 配置
3. 启动前端开发服务器
4. 启动后端服务
5. 测试前端访问后端 API
6. 验证所有功能正常
