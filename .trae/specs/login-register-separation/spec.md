# 登录注册页面功能分离与优化 - 产品需求文档

## Overview
- **Summary**: 基于现有前端项目已实现的通用登录页和注册页，进行功能优化与调整，明确区分用户端和商家端的登录注册入口，避免用户混淆。
- **Purpose**: 通过清晰的入口分离和功能明确化，提升用户体验，减少用户在登录注册流程中的困惑。
- **Target Users**: 平台普通用户、商家用户

## Goals
- 在通用登录页添加明确的"商家登录"跳转入口
- 从通用注册页移除所有商家注册相关选项和引导
- 确保商家注册功能仅在专门的商家注册页面提供
- 保持整体UI风格一致性
- 确保响应式设计

## Non-Goals (Out of Scope)
- 不修改后端API接口
- 不改变现有的路由结构
- 不修改商家注册页面的功能逻辑
- 不添加新的业务功能

## Background & Context
- 当前通用登录页（`/login`）包含角色选择（用户/商家/管理员），导致用户混淆
- 当前通用注册页（`/register`）需要移除商家注册相关引导
- 商家登录页（`/merchant/login`）和商家注册页（`/merchant/register`）已实现独立页面
- 需要明确区分用户端和商家端的入口

## Functional Requirements
- **FR-1**: 通用登录页添加商家登录入口
  - 在登录表单底部添加"商家登录"跳转链接
  - 链接应具有清晰的视觉标识（如店铺图标、不同颜色等）
  - 点击后平滑导航至商家登录页面（`/merchant/login`）
  - 移除原有的角色选择功能（用户/商家/管理员）
  - 通用登录页仅保留普通用户登录功能

- **FR-2**: 通用注册页移除商家注册选项
  - 移除所有与"商家注册"相关的选项、入口及引导性文字
  - 确保通用注册页仅保留普通用户注册功能
  - 在注册页底部添加"商家入驻"跳转链接（可选，视设计而定）
  - 保持普通用户注册流程的完整性和正确性

- **FR-3**: 商家注册功能明确化
  - 确保商家注册功能仅在`/merchant/register`页面提供
  - 验证商家注册页面的独立性和功能完整性
  - 商家注册页面应包含返回用户端登录的入口

## Non-Functional Requirements
- **NFR-1**: 视觉设计一致性
  - 所有页面应采用统一的设计风格
  - 颜色方案、字体、间距等视觉元素应保持一致
  - 响应式设计，适配不同屏幕尺寸

- **NFR-2**: 用户体验优化
  - 入口清晰明确，避免用户混淆
  - 页面交互流畅、直观
  - 跳转过渡平滑

## Constraints
- **Technical**: Vue 3 + TypeScript + Element Plus 技术栈
- **Business**: 保持现有功能逻辑不变
- **Dependencies**: 依赖 Element Plus 组件库

## Assumptions
- 现有后端API接口保持不变
- 现有路由结构保持不变
- 现有商家登录/注册页面功能正常

## Acceptance Criteria

### AC-1: 通用登录页商家入口
- **Given**: 用户访问 `/login` 页面
- **When**: 页面加载完成
- **Then**: 页面应显示普通用户登录表单，底部有明确的"商家登录"跳转入口
- **Verification**: `human-judgment`

### AC-2: 通用登录页功能正常
- **Given**: 用户在通用登录页输入信息
- **When**: 用户提交表单
- **Then**: 仅支持普通用户登录，不再支持角色选择
- **Verification**: `programmatic`

### AC-3: 通用注册页无商家选项
- **Given**: 用户访问 `/register` 页面
- **When**: 页面加载完成
- **Then**: 页面仅显示普通用户注册表单，无任何商家注册相关选项或引导
- **Verification**: `human-judgment`

### AC-4: 商家注册页面独立
- **Given**: 用户访问 `/merchant/register` 页面
- **When**: 页面加载完成
- **Then**: 页面应显示完整的商家注册表单，功能独立且完整
- **Verification**: `programmatic`

### AC-5: 响应式设计
- **Given**: 用户在不同设备上访问页面
- **When**: 页面加载完成
- **Then**: 页面应适配不同屏幕尺寸，保持良好的用户体验
- **Verification**: `human-judgment`
