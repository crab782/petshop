# 手机号认证系统 - 产品需求文档

## Overview
- **Summary**: 修改用户注册和登录逻辑，将手机号作为唯一标识，允许用户名和邮箱重复，登录改为手机号登录。
- **Purpose**: 简化用户认证流程，降低注册门槛，提升用户体验。
- **Target Users**: 平台用户

## Goals
- 将手机号作为用户唯一标识，不允许重复
- 允许用户名和邮箱重复，不做唯一性检查
- 修改登录逻辑为手机号登录，不再支持用户名/邮箱登录
- 保持现有功能逻辑不变，只修改认证方式

## Non-Goals (Out of Scope)
- 不修改商家端的注册和登录逻辑
- 不修改管理员端的认证逻辑
- 不添加新的业务功能
- 不修改数据库表结构

## Background & Context
- 当前注册接口要求用户名、邮箱、手机号都唯一，导致注册门槛较高
- 用户可能忘记用户名或邮箱，但更容易记住手机号
- 手机号作为唯一标识更符合国内用户习惯
- 后端API接口已实现，需要修改验证逻辑

## Functional Requirements
- **FR-1**: 修改注册逻辑
  - 手机号作为唯一标识，不允许重复
  - 用户名允许重复，不做唯一性检查
  - 邮箱允许重复，不做唯一性检查
  - 保留用户名和邮箱字段，但降低其重要性

- **FR-2**: 修改登录逻辑
  - 登录方式改为手机号+密码
  - 不再支持用户名登录
  - 不再支持邮箱登录
  - 保持密码验证逻辑不变

- **FR-3**: 修改前端页面
  - 注册页面：调整表单提示，强调手机号为必填且唯一
  - 登录页面：修改为手机号登录，移除用户名/邮箱选项

## Non-Functional Requirements
- **NFR-1**: 数据一致性
  - 确保现有用户数据不受影响
  - 确保手机号唯一性约束生效

- **NFR-2**: 用户体验
  - 登录流程更简单，只需记住手机号
  - 注册流程更灵活，用户名和邮箱可随意填写

- **NFR-3**: 安全性
  - 手机号验证规则保持严格
  - 密码验证逻辑保持不变

## Constraints
- **Technical**: Spring Boot + MyBatis-Plus + Vue 3技术栈
- **Business**: 保持现有功能逻辑不变
- **Dependencies**: 依赖现有数据库表结构

## Assumptions
- 现有用户数据中手机号已唯一
- 用户能够接受手机号作为唯一登录方式
- 后端API接口路径保持不变

## Acceptance Criteria

### AC-1: 注册逻辑修改
- **Given**: 用户提交注册信息
- **When**: 手机号已存在
- **Then**: 系统应返回错误提示"手机号已被注册"
- **Verification**: `programmatic`

### AC-2: 注册逻辑修改（用户名重复）
- **Given**: 用户提交注册信息
- **When**: 用户名已存在但手机号不同
- **Then**: 系统应允许注册成功
- **Verification**: `programmatic`

### AC-3: 注册逻辑修改（邮箱重复）
- **Given**: 用户提交注册信息
- **When**: 邮箱已存在但手机号不同
- **Then**: 系统应允许注册成功
- **Verification**: `programmatic`

### AC-4: 登录逻辑修改
- **Given**: 用户使用手机号和密码登录
- **When**: 手机号和密码正确
- **Then**: 系统应返回登录成功
- **Verification**: `programmatic`

### AC-5: 登录逻辑修改（用户名登录）
- **Given**: 用户使用用户名和密码登录
- **When**: 提交登录请求
- **Then**: 系统应返回错误提示"请使用手机号登录"
- **Verification**: `programmatic`

### AC-6: 前端注册页面修改
- **Given**: 用户访问注册页面
- **When**: 页面加载完成
- **Then**: 页面应显示手机号为必填项，并提示手机号为唯一标识
- **Verification**: `human-judgment`

### AC-7: 前端登录页面修改
- **Given**: 用户访问登录页面
- **When**: 页面加载完成
- **Then**: 页面应显示手机号输入框，不再显示用户名/邮箱输入框
- **Verification**: `human-judgment`
