## 失败原因分析

### 1. 返回 500 错误的测试
这些测试失败的原因可能是：
- 后端接口实现存在异常处理问题
- 数据库操作异常
- 业务逻辑错误

建议检查：
- `MerchantApiController` 中的 `updateService`、`deleteService`、`getServiceById` 方法
- `MerchantApiController` 中的 `updateProduct`、`deleteProduct`、`getProductById`、`updateProductStatus` 方法
- `MerchantApiController` 中的 `updateOrderStatus`、`updateLogisticsInfo` 方法
- `MerchantApiController` 中的 `getSettings` 方法

### 2. 权限验证测试失败
- `/api/merchant/**` 路径需要认证，返回 401
- SecurityConfig 中可能已修改权限配置
