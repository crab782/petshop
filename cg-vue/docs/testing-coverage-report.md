## 主要问题分析

### 1. 测试失败问题

当前有 260 个测试用例失败，主要集中在以下模块：

- `src/views/user/user-shop/` - 店铺详情页面测试失败
- 其他用户端和商家端页面测试

失败原因分析：
- Mock 配置问题
- 异步操作未正确等待
- 组件渲染问题
- 数据结构不匹配

### 2. 覆盖率低的原因

1. **测试失败**：失败的测试导致覆盖率统计不准确
2. **Mock 配置**：部分 Mock 可能阻止了实际代码的执行
3. **未测试的模块**：部分模块缺少测试用例

## 测试策略建议

### 1. Mock 策略调整

- 只 Mock 外部依赖（API、第三方库）
- 避免过度 Mock 导致代码未执行
- 使用真实的 Store 和 Router

### 2. 测试优先级

**高优先级**：
- 核心业务逻辑
- 数据处理和验证
- 用户关键操作流程

**中优先级**：
- 页面组件
- 公共组件
- 工具函数

**低优先级**：
- UI 展示组件
- 配置文件
- 类型定义文件

### 3. 测试类型分布

建议的测试类型分布：

- 单元测试：70%
- 集成测试：20%
- E2E 测试：10%
